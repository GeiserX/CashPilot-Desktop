# CLAUDE.md — CashPilot Desktop

## Overview

Cross-platform desktop app (Wails 2.x + Go + vanilla TypeScript) for deploying and monitoring passive-income and DePIN services via Docker-compatible runtimes.

## Tech Stack

- **Backend**: Go 1.26.x (Wails framework)
- **Frontend**: Vanilla TypeScript + Vite (no UI framework; single-file innerHTML templating in `frontend/src/main.ts`)
- **Storage**: SQLite (local state)
- **Runtime**: Docker socket / Podman / managed VM
- **Build**: Wails CLI (`wails build`)

## Development

```bash
wails dev          # hot-reload dev mode
wails build        # production build
go test ./...      # run Go tests
```

## Architecture

- `main.go` / `app.go` — Wails app bootstrap
- `internal/` — core business logic (container management, collectors, state)
- `services/` — service catalog definitions (passive income providers)
- `frontend/` — vanilla TypeScript SPA (communicates via Wails bindings)
- `internal/exchange/` — FX rates (crypto + fiat → display currency) powering the earnings summary
- `fleet_server.go` — token-authenticated worker/mobile heartbeat API (loopback by default)

## Key Rules

- NEVER hardcode credentials — the credential master key lives in the OS keychain via `internal/config` (`config.MasterKey`, using `github.com/zalando/go-keyring`); credentials are AES-256-GCM-encrypted in `internal/store`. There is no `internal/keyring` package.
- Container operations MUST handle both Docker and Podman runtimes
- All external container images: pinned semver tags, never `:latest`
- SQLite migrations are forward-only (no down migrations)
- Frontend Wails bindings auto-generated — do not edit `frontend/wailsjs/` manually

## Testing

```bash
go test -race -coverprofile=coverage.out ./...
go tool cover -html=coverage.out
```

## Build & Release

Tag releases (`v*`) build Linux/macOS/Windows on free GitHub-hosted runners
(`ubuntu-latest`/`macos-latest`/`windows-latest`). Installers are currently UNSIGNED
(no macOS codesign/notarization; Windows signing only if `WINDOWS_SIGN_CERT` is set). Signing is planned.
